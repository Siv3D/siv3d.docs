# 3. Siv3D の基本
Siv3D プログラムの基本的な構成について学びます。

## 3.1 インクルードするヘッダ
- 基本的には、たった 1 行のコードで OK です：

```cpp
# include <Siv3D.hpp>
```

- この 1 行を入れるだけで、必要な機能のほとんどが使えるようになります
- Siv3D.hpp の中で `<algorithm>` や `<vector>` など、主要な標準ライブラリのヘッダファイルもインクルードされています

### 特徴的なところ
- 通常の C++ プログラミングで必要な標準ライブラリのヘッダファイルが不要です
- Siv3D は必要な機能を 1 つのヘッダファイルにまとめることで、プログラミングの準備を簡単にします


## 3.2 プログラムの始まり方
- 通常の C++ では `int main()` から始まりますが、Siv3D では代わりに `void Main()` 関数を書きます

```cpp
# include <Siv3D.hpp>

void Main()
{
	// ここにプログラムを書く
}
```

- Siv3D の内部プログラムが次のような構成になっているためです：
	- ① プログラムが始まる前に、Siv3D の内部プログラムが必要な初期化を全て実行する
	- ② そのあとで開発者の書いた `Main()` を実行する
	- ③ `Main()` が終わったら、Siv3D の内部プログラムが自動的に後片付けを実行する
- つまり、開発者は面倒な準備や後片付けを気にせず、やりたいことだけを `Main()` 関数の中に書けば OK ということです
- レストランの調理場で例えると：
	- スタッフがあらかじめ調理器具を用意してくれる（初期化）
	- シェフは調理だけに集中できる（`Main` 関数）
	- 終わったら、スタッフが後片付けをしてくれる（終了処理）
- というような感じです


```C++ title=" 簡略化した Siv3D の内部プログラム"
int main()
{
	Siv3D初期化(); // 必要な初期化処理を行う

	Main(); // 開発者が書いたプログラムを実行する

	Siv3D終了処理(); // 後片付けを行う
}
```

!!! info "既存のプログラムの中で Siv3D を使うことはできない"
	- Siv3D は内部に `main()` 関数を持っているため、`main()` 関数に書かれた既存のプログラム（過去に作ったゲームプログラムなど）の中で新しく Siv3D を使うことはできません
	

## 3.3 最小の Siv3D プログラム
- 手元のエディタで Siv3D の最小のプログラムを書いて、ビルド・実行してみましょう

```cpp title="最小の Siv3D プログラム"
# include <Siv3D.hpp>

void Main()
{

}
```

- この `Main` 関数は何もすることがないため、実行すると一瞬で終了します
- ウィンドウも表示されず、何も起こっていないように見えるでしょう


## 3.4 メインループを書いてウィンドウを表示し続ける
- プログラムをすぐに終了させないためには、**メインループ**を書く必要があります
- 次のコードのハイライト部分 `while (System::Update()){ }` がメインループです
- `while` 文によって、ハイライト部分のプログラムが半永久的に繰り返されます

```cpp title="メインループを追加したプログラム" hl_lines="5-8"
# include <Siv3D.hpp>

void Main()
{
	while (System::Update())
	{
		// ここにプログラムを書く
	}
}
```

- 繰り返しのたびに、`System::Update()` はウィンドウの表示や音楽の再生、マウスやキーボードの入力情報などを更新します
- グラフィックスの表示やユーザの入力の取得などを継続的に処理できるようになります
- メインループが回っている間は、ウィンドウが表示され続けます

## 3.5 プログラムの終了方法
- `System::Update()` は、普段は `true` を返すため、半永久的にメインループが続きます
- 特定の操作をすると、`System::Update()` は `false` を返すようになり、メインループを終了させます
- メインループを抜けて、そのまま `Main` 関数の終端まで到達するとプログラムが終了します

```cpp title="メインループから抜けるとプログラムが終了する" hl_lines="6"
# include <Siv3D.hpp>

void Main()
{
	// System::Update() が false を返すまでメインループを繰り返す
	while (System::Update())
	{

	}
}
```

- `System::Update()` が `false` を返すようにする方法はいくつかあります
	1. **ウィンドウの閉じるボタンを押す**
	2. **++esc++ キーを押す**
	3. **プログラムの中で `System::Exit()` を呼び出す**
- a, b の方法は、実際にウィンドウを閉じるか、++esc++ キーを押すことで確認できます
- c の方法を試すコードは次のとおりです
	- プログラムの開始から 5 秒経過したら `System::Exit()` を呼び出し、それ以降の `System::Update()` が `false` を返すようになります
	- つまり、5 秒経過でメインループが自動的に終了し、プログラムが終了します

```cpp title="プログラムの開始から 5 秒経過したらプログラムを終了する" hl_lines="11"
# include <Siv3D.hpp>

void Main()
{
	while (System::Update())
	{
		// プログラムの開始から 5 秒経過したら
		if (5.0 <= Scene::Time())
		{
			// プログラムの終了を指示する
			System::Exit();
		}
	}
}
```

- `System::Exit()` を呼ぶと、そこで直ちにプログラムが終了するわけではありません
- あくまで「次の `System::Update()` が `false` を返すようにする」という指示を出すだけです


## 3.6 Main 関数からの return
- 3.5 の方法とは別に、`Main` 関数から `return` することでプログラムを終了させることもできます
- とくに、`System::Update()` を待たず、直ちにプログラムを終了させたい場合に有用です
- タブを切り替えて、`System::Exit()` と `return` の違いを確認してみましょう

=== "3.5 の方法による終了"

	`System::Exit()` 呼び出し後、**`処理 A`, `処理 B` が実行されてから** `Main` 関数が終了します。

	```C++ hl_lines="10 13 16"
	# include <Siv3D.hpp>

	void Main()
	{
		while (System::Update())
		{
			// プログラムの開始から 5 秒経過したら
			if (5.0 <= Scene::Time())
			{
				System::Exit(); // 次の System::Update() が false を返すようにする
			}

			処理A();
		}

		処理B();
	}
	```

=== "3.6 の方法による終了"

	`return;` することで、**`処理 A`, `処理 B` は実行されず** 直ちに `Main` 関数が終了します。

	```C++ hl_lines="10"
	# include <Siv3D.hpp>

	void Main()
	{
		while (System::Update())
		{
			// プログラムの開始から 5 秒経過したら
			if (5.0 <= Scene::Time())
			{
				return; // ここで直ちに Main() が終了する
			}

			処理A();
		}

		処理B();
	}
	```

- どちらの方法を使うかどうかは、プログラムの構造や目的によります
- Siv3D の公式サンプルは、すべて 3.5 の方法でプログラムを終了しています
	- 「処理 B」の場所にゲームのセーブ処理などを記述できて都合が良いためです
- 終了方法をカスタマイズする方法（++esc++ を押しても終了しないようにするなど）は、この先のチュートリアルで学びます


!!! info "System::Exit() は必須ではない"
	- `System::Exit()` は「後片付けを行う」関数ではありません
	- `System::Exit()` は「プログラムの終了指示を出す」だけの関数です。使用は必須ではありません
	- ++esc++ キーを押した場合や、ウィンドウの閉じるボタンを押した場合、`return;` で終了した場合でも、いずれもプログラムは正常に終了します
